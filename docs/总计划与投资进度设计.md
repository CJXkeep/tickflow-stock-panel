# 总计划与长期目标规划设计

本文档定义"总计划 / 长期目标规划"功能的产品边界、领域模型和第一版实现范围。这里的总计划不是普通资产看板,而是把类似 [10年做到150万投资目标规划.md](./计划/10年做到150万投资目标规划.md) 的长期目标、投入情景、年度里程碑、阶段执行计划、风险规则和投资进度放进系统。

## 1. 定位

总计划是系统的投资总控台:

```text
10 年目标
  -> 目标规划与投入情景
  -> 年度里程碑与阶段执行计划
  -> 当前可投资资产
  -> 持仓与现金结构
  -> 目标曲线与当前进度
  -> 风险档案与静默健康度
  -> 历史资产快照与年度复盘
```

它回答的问题是:

- 当前离长期目标还有多远。
- 当前目标在不同月投入假设下需要多高年化。
- 当前进度相对年度里程碑是领先、正常还是落后。
- 当前资产增长来自净投入还是投资收益。
- 当前仓位结构是否还符合自己的风险档案。
- 当前难度相比原始计划是降低了还是升高了。
- 哪些资产计入目标,哪些只是备用金或暂不纳入系统。

它不回答的问题是:

- 今天应该买哪只股票。
- 是否应该立刻卖出某只持仓。
- 自动接管持仓调整。
- 做完整家庭净资产负债表。
- 做完整证券流水记账和税费核算。

## 2. 与其他模块的关系

总计划位于系统最高层,观察池、策略、复盘、监控都在它下面工作。

```text
InvestmentMasterPlan
  -> LongTermGoal / PlanningScenario / Milestone
  -> Portfolio / Position
  -> AnnualReview
  -> ObservationItem
  -> DailyTradingPlan
  -> Review / Strategy / Monitor
```

推荐导航:

```text
总计划
复盘
观察池
策略
回测
监控
数据
```

复盘页可以展示总计划摘要,但总计划应作为一级页面存在。复盘页回答"今天市场和机会怎么样",总计划页回答"我离长期目标还有多远,当前资产结构是否仍在自己的节奏内"。

## 3. 第一版原则

第一版只做动态进度和静默状态,不做主动干预。

- 持仓、现金、资产分类由用户手动维护。
- 系统用行情估算持仓市值。
- 当前持仓和现金明细是权威源。
- 资产快照由系统派生和保存,不作为主要手工入口。
- 长期目标规划可以由模板初始化,例如"10 年做到 150 万"。
- 月投入情景、年度里程碑和阶段执行计划是规划对象,不是静态 Markdown 文本。
- 保留已发生净投入记录,不要求预设未来每月投入计划。
- 显示状态和健康度,不弹提醒,不生成任务,不强制阻断。
- 风险档案可自定义,默认规则只是起点。
- 总计划不做荐股,只提供风险坐标系。

## 4. 核心概念

### 4.1 总计划

`InvestmentPlan` 表示一个长期目标,例如"10 年做到 150 万"。

关键字段:

```text
id
name
status: active | archived
current_version_id
created_at
updated_at
```

当前系统可以先只支持一个 active 总计划,但模型上保留多计划可能性。

### 4.2 计划版本

长期目标可以重设,但不能无痕覆盖。每次调整目标金额、目标日期、起始资产或基准投入假设时,都生成一个 `PlanVersion`。

```text
PlanVersion
  id
  plan_id
  version_no
  target_amount
  start_date
  target_date
  initial_plan_asset
  baseline_monthly_contribution
  baseline_required_annual_return
  risk_profile_id
  reason
  created_at
```

当前页面默认展示当前版本,并提供版本历史:

```text
v1: 2026-06, 10 年 150 万, 起始 10 万, 基准月投 1000
v2: 2028-01, 目标不变, 起始资产和基准假设调整
```

### 4.3 规划情景

`PlanScenario` 表示同一目标下的不同投入假设,用于回答"如果月投提高,目标难度下降多少"。

```text
PlanScenario
  id
  plan_version_id
  label
  monthly_contribution
  required_annual_return
  required_monthly_return
  note
```

默认可由 `PlanVersion` 派生以下情景:

```text
月投 1000 -> 所需年化约 26.2%
月投 2000 -> 所需年化约 22.2%
月投 3000 -> 所需年化约 18.7%
月投 5000 -> 所需年化约 12.9%
```

情景不是承诺未来一定投入,只是帮助用户理解目标难度。

### 4.4 年度里程碑

`PlanMilestone` 表示长期目标拆到年度或阶段节点后的目标资产。

```text
PlanMilestone
  id
  plan_version_id
  year_no
  target_date
  target_asset
  note
```

例如"10 年做到 150 万"可以生成:

```text
第 1 年 14 万
第 3 年 25 万
第 5 年 43 万
第 8 年 92 万
第 10 年 150 万
```

系统用里程碑判断当前进度:

```text
当前计划资产 >= 当前里程碑目标 * 1.05 -> 领先
当前计划资产 >= 当前里程碑目标 * 0.9  -> 正常
否则 -> 落后
```

阈值第一版可以写死,后续再放入用户配置。

### 4.5 阶段执行计划

`PlanStage` 表示长期规划里的阶段重点,例如第 1-3 年积累本金和能力、第 4-6 年扩大复利基础、第 7-10 年防止大回撤毁掉成果。

```text
PlanStage
  id
  plan_version_id
  start_year
  end_year
  title
  focus_items
  target_notes
  risk_notes
```

阶段执行计划只作为静默参考,不生成强提醒、不阻断交易研究流程。

### 4.6 风险档案

`RiskProfile` 不是系统强加的风险风格,而是用户自定义边界。激进可以,但要知道自己正在激进。

```text
RiskProfile
  id
  name
  core_min_pct
  core_max_pct
  aggressive_min_pct
  aggressive_max_pct
  cash_min_pct
  cash_max_pct
  single_position_warn_pct
  drawdown_watch_pct
  drawdown_high_pct
```

第一版可预置稳健型、平衡型、进取型,但用户可以保存为自己的风险档案。系统只按用户当前档案展示偏离状态,不做道德审判式提示。

### 4.7 资产口径

10 年目标只统计可投资金融资产,不做完整净资产系统。

资产口径:

```text
plan_asset      计入长期目标
reserve_asset   生活备用金,展示但不计入目标
ignored_asset   暂不纳入系统
```

第一版纳入范围:

- 股票
- 基金
- 现金
- 货币基金
- 其他可投资金融资产

第一版不纳入:

- 房产
- 车辆
- 社保
- 公积金
- 复杂负债

页面可以同时展示:

```text
计划资产: 128000 / 1500000
备用金: 30000
总金融资产: 158000
```

### 4.8 持仓

`Position` 是当前资产明细,也是总资产计算的权威源。持仓由用户手动调整,系统只估值和归档。

```text
Position
  id
  symbol
  name
  asset_type: stock | fund | cash | other
  asset_scope: plan_asset | reserve_asset | ignored_asset
  asset_bucket: core | aggressive | cash | unknown
  shares
  cost_price
  manual_price
  manual_market_value
  note
  updated_at
```

估值规则:

```text
有 symbol 且行情可用:
  market_value = shares * latest_price

无 symbol 或行情不可用:
  market_value = manual_market_value
  或 manual_price * shares
```

`asset_bucket` 是用户视角的持有逻辑,不是资产天然属性。系统可以猜测,但第一版应让用户确认。

### 4.9 净投入记录

第一版不要求未来投入计划,但需要记录已发生净投入,否则无法区分资产增长来自转入还是收益。

```text
ContributionRecord
  id
  date
  amount
  note
  created_at
```

约定:

- 正数表示转入。
- 负数表示转出。
- 不做复杂现金流水。
- 不要求每笔交易都对应投入记录。

### 4.10 持仓调整日志

持仓调整由用户完成,系统保存轻量日志用于追溯,不做会计记账。

```text
PositionAdjustmentLog
  id
  position_id
  adjusted_at
  before_snapshot
  after_snapshot
  reason: manual_update | buy | sell | rebalance | correction
  note
```

用途:

- 年度复盘时知道资产结构变化原因。
- 排查资产曲线异常。
- 不用于成本核算、收益归因、税费计算。

### 4.11 资产快照

`PortfolioSnapshot` 是系统根据当前持仓、现金和行情派生出的历史记录。

```text
PortfolioSnapshot
  date
  plan_asset
  reserve_asset
  total_financial_asset
  market_value
  cash_value
  total_contribution
  investment_profit
  drawdown_from_peak
  source: auto | manual_adjustment
  created_at
```

原则:

- 当前持仓和现金明细是权威源。
- 快照是派生历史,不作为主要手工入口。
- 同一天重复生成时更新当天快照。
- 如需校准总资产,应调整持仓、现金或新增其他资产项,不要直接改历史快照。

### 4.12 年度复盘

`AnnualReview` 表示长期规划每年的复盘记录。它来自规划模板,但最终由用户填写。

```text
AnnualReview
  id
  plan_id
  year
  review_date
  current_asset
  annual_contribution
  annual_return_rate
  milestone_gap
  core_pct
  aggressive_pct
  cash_pct
  wins
  mistakes
  monthly_contribution_adjustment
  allocation_adjustment
  next_year_focus
  created_at
  updated_at
```

年度复盘不替代交易流水或专业业绩归因,只是把长期规划从"写过一次"变成"每年会回看"。

## 5. 进度计算

### 5.1 总资产与收益

```text
计划资产 = sum(position.market_value where asset_scope = plan_asset)
备用资产 = sum(position.market_value where asset_scope = reserve_asset)
总金融资产 = 计划资产 + 备用资产
累计净投入 = sum(contribution.amount)
累计投资收益 = 计划资产 - 累计净投入
简化累计收益率 = 累计投资收益 / 累计净投入
目标完成度 = 计划资产 / target_amount
年度里程碑差额 = 计划资产 - 当前年度 milestone.target_asset
```

第一版使用简化收益率,不做 XIRR、时间加权收益、分红税费归因。后续如果需要,再引入专业业绩计算。

### 5.2 原始基准曲线

原始基准曲线来自当前 `PlanVersion`:

```text
initial_plan_asset
baseline_monthly_contribution
target_amount
start_date
target_date
baseline_required_annual_return
```

用途:

- 保留最初计划的参照。
- 展示当前资产相对原计划领先还是落后。
- 生成月度和年度目标点。

### 5.3 里程碑状态

长期规划页需要把当前计划资产映射到当前年度里程碑:

```text
领先: 当前计划资产 >= 当前里程碑目标 * 1.05
正常: 当前计划资产 >= 当前里程碑目标 * 0.9
落后: 当前计划资产 < 当前里程碑目标 * 0.9
```

如果落后,页面只展示状态和可能的调整方向,例如"提高月投入、延长周期、降低目标或复盘收益假设",不自动给交易建议。

### 5.4 当前重算难度

在不要求用户填写未来投入计划的前提下,总计划页展示两种当前所需收益率:

```text
无未来新增投入:
  从当前计划资产到目标日期达到 target_amount 所需年化

沿用原始基准投入:
  从当前计划资产出发,未来仍按 baseline_monthly_contribution 投入,到目标日期达到 target_amount 所需年化
```

主口径是"无未来新增投入"压力测试,辅口径是"沿用原始基准投入"。这两个数告诉用户从现在开始目标难度是升高还是降低,但不强迫用户承诺未来每月投入。

### 5.5 静默健康度

第一版只在总计划页展示状态,不弹窗、不红点、不生成任务。

状态维度:

```text
进度: 领先 | 正常 | 落后
收益: 正收益 | 持平 | 亏损
仓位: 在范围内 | 接近边界 | 超出边界
回撤: 正常 | 关注 | 高回撤
数据: 完整 | 待分类 | 估值缺失
```

总健康度可以派生为:

```text
正常
关注
偏离
高风险
```

语气要克制。例如:

```text
当前激进仓 48%, 接近你设置的 50% 上限。
```

不要写成:

```text
你太激进了,必须降仓。
```

## 6. 与观察池和策略的连接

总计划不限制观察池和策略,只提供风险坐标系。

观察项可增加:

```text
intended_bucket: core | aggressive | cash | unknown
planned_capital
risk_note
```

系统可以静默展示:

```text
当前激进仓占比: 38%
观察池计划激进投入: 12%
如果全部执行,激进仓可能达到 50%
```

但不禁止加入观察池,不禁止运行策略,不生成交易指令。

## 7. 页面结构

总计划页面建议包含:

1. 顶部目标进度
   - 计划资产 / 目标资产
   - 距离目标金额
   - 目标完成度
   - 当前所需年化双口径

2. 长期目标规划
   - 目标金额、目标日期、初始本金、基准月投
   - 月投入情景表
   - 年度里程碑
   - 阶段执行计划
   - 风险控制规则

3. 资产拆解
   - 计划资产
   - 备用资产
   - 总金融资产
   - 累计净投入
   - 简化投资收益

4. 目标曲线
   - 原始基准曲线
   - 实际资产快照曲线
   - 年度里程碑

5. 持仓结构
   - core / aggressive / cash / unknown 占比
   - 单一持仓占比
   - 估值缺失项

6. 持仓维护
   - 当前持仓列表
   - 现金/其他资产项
   - 分类和资产口径编辑
   - 持仓调整日志

7. 净投入记录
   - 转入/转出记录
   - 累计净投入

8. 年度复盘
   - 年度复盘模板
   - 历史年度复盘记录
   - 下一年重点

9. 计划设置
   - 当前计划版本
   - 风险档案
   - 版本历史

复盘 Dashboard 摘要只展示:

```text
计划资产 / 本月基准目标 / 年度目标
累计净投入 / 简化投资收益
仓位结构摘要
静默健康度
```

## 10. 第一版范围

必须做:

- 总计划一级页面。
- 当前计划版本和目标设置。
- 长期规划模板初始化,至少支持"10 年做到 150 万"。
- 月投入情景表。
- 年度里程碑。
- 阶段执行计划。
- 风险控制规则展示。
- 持仓/现金/其他资产维护。
- 资产口径: `plan_asset` / `reserve_asset` / `ignored_asset`。
- 持仓分类: `core` / `aggressive` / `cash` / `unknown`。
- 行情可用时自动估值,不可用时使用手动估值。
- 已发生净投入记录。
- 派生资产快照。
- 原始基准曲线。
- 当前所需年化双口径。
- 当前进度相对年度里程碑的状态。
- 年度复盘模板和年度复盘记录。
- 静默健康度。
- 计划版本历史。

暂不做:

- 证券账户自动接入。
- 自动交易或交易指令。
- 完整交易流水和会计记账。
- XIRR / TWR 等复杂业绩归因。
- 主动提醒、任务、红点、弹窗。
- 完整家庭资产负债表。
- 自动替用户判断风险风格。

## 11. 验收标准

第一版完成后,用户应能:

1. 创建一个"10 年 150 万"总计划。
2. 看到 1000 / 2000 / 3000 / 5000 月投入情景下的目标难度。
3. 看到第 1 年到第 10 年的年度里程碑。
4. 看到第 1-3 年、第 4-6 年、第 7-10 年的阶段执行重点。
5. 录入当前持仓、现金和其他可投资资产。
6. 手动调整持仓,并保留轻量调整日志。
7. 标记每个资产是否计入目标、是否属于备用金。
8. 标记每个持仓属于核心仓、激进仓、现金仓或未知。
9. 录入已发生净投入和转出。
10. 看到当前计划资产、备用资产、总金融资产。
11. 看到累计净投入、简化投资收益和目标完成度。
12. 看到当前进度相对年度里程碑的领先、正常或落后状态。
13. 看到原始基准曲线与实际资产快照曲线。
14. 看到从当前状态到目标日所需年化的双口径。
15. 填写一次年度复盘记录。
16. 看到静默健康度,但不会被系统打扰。
17. 重设目标时保留版本历史。

满足这些标准后,总计划就能从静态 Markdown 目标,升级为系统里的长期投资导航层。

## 12. 执行拆分

当前代码结构已经支持执行这份计划。后端已有 `data/user_data` 本地持久化、JSON/Parquet 混合存储、FastAPI router、Polars 数据读写模式;前端已有一级路由、左侧导航、TanStack Query 和页面组件模式。第一版不需要引入新数据库,也不需要改行情/策略/回测主链路。

建议按以下顺序落地。

### Phase 1: 后端最小闭环

目标:先让总计划数据可保存、可汇总、可被前端读取。

新增文件:

```text
backend/app/services/investment_plan.py
backend/app/api/investment_plan.py
```

接入点:

```text
backend/app/main.py include_router(investment_plan.router)
```

实现范围:

- `InvestmentPlan` 当前版本读取与保存。
- "10 年做到 150 万"模板初始化。
- `PlanScenario` 读取、保存和默认生成。
- `PlanMilestone` 读取、保存和默认生成。
- `PlanStage` 读取、保存和默认生成。
- 默认风险档案读取与保存。
- `Position` 增删改查。
- `ContributionRecord` 增删查。
- `PositionAdjustmentLog` 自动追加。
- `PortfolioSnapshot` 当日派生与保存。
- `AnnualReview` 增改查。
- `/api/investment-plan/summary` 聚合计算。

后端第一版可以先使用:

```text
investment_plan.json
investment_plan_versions.jsonl
risk_profiles.json
plan_scenarios.json
plan_milestones.json
plan_stages.json
positions.parquet
contributions.parquet
position_adjustments.jsonl
portfolio_snapshots.parquet
annual_reviews.jsonl
```

验收:

- 空数据时能返回默认计划草稿和空 summary,页面不报错。
- 可以从"10 年做到 150 万"模板初始化计划。
- 初始化后能返回月投入情景、年度里程碑和阶段执行计划。
- 新增一个现金资产后,summary 能计算计划资产。
- 新增一只 A 股持仓后,有行情时使用行情估值,无行情时使用手动估值。
- 新增净投入后,summary 能区分累计净投入和简化投资收益。
- summary 能返回当前年度里程碑状态。
- 能保存一条年度复盘记录。
- 编辑持仓后自动写一条调整日志。

### Phase 2: 前端一级页面

目标:让用户能维护数据并看到总计划进度。

新增文件:

```text
frontend/src/pages/InvestmentPlan.tsx
```

修改文件:

```text
frontend/src/router.tsx
frontend/src/components/Layout.tsx
frontend/src/lib/api.ts
frontend/src/lib/queryKeys.ts
```

实现范围:

- 左侧导航新增"总计划"。
- `/investment-plan` 路由。
- 顶部目标进度区。
- 目标规划参数区。
- 月投入情景区。
- 年度里程碑区。
- 阶段执行计划区。
- 资产拆解区。
- 持仓结构区。
- 持仓维护表单。
- 净投入记录表单。
- 年度复盘模板和填写区。
- 静默健康度展示。

暂缓:

- 复杂图表交互。
- 版本历史编辑 UI 的完整体验。
- 观察池 planned_capital 联动。
- Dashboard 摘要。

验收:

- 用户能在页面新增/编辑/删除持仓。
- 用户能载入"10 年做到 150 万"模板。
- 用户能看到月投入情景和年度里程碑。
- 用户能录入现金和其他资产。
- 用户能录入净投入。
- 用户能填写年度复盘。
- 页面刷新后数据仍存在。
- 修改数据后 summary 即时刷新。

### Phase 3: 目标曲线与快照

目标:让总计划从静态汇总变成长期轨迹。

实现范围:

- 原始基准曲线生成。
- 实际资产快照曲线读取。
- 当前所需年化双口径。
- 月投入情景的所需年化计算。
- 当前年度里程碑差额和状态。
- 当日快照重建按钮。
- 年度里程碑展示。

验收:

- 有初始计划和当前资产时,能看到原始月度目标点。
- 不同月投入情景能显示不同所需年化。
- 当前资产能映射到当前年度里程碑状态。
- 能看到无未来新增投入与沿用基准投入两种所需年化。
- 修改持仓后,重建当日快照会更新实际资产曲线。

### Phase 4: 系统挂接

目标:让总计划成为系统顶层,但不干扰交易研究流程。

实现范围:

- 复盘 Dashboard 增加总计划摘要。
- 观察项增加 `intended_bucket`、`planned_capital`、`risk_note` 字段。
- 观察池页面显示计划风险上下文。
- 不做主动提醒、不做任务、不做红点。

验收:

- Dashboard 能看到总计划摘要。
- 观察项能标记计划投入和计划仓位分类。
- 总计划页能静默展示观察池潜在投入对仓位结构的影响。

### 暂不执行项

以下内容先不进入第一轮实现:

- 券商账户接入。
- 自动交易。
- 完整交易流水。
- 专业收益归因。
- 家庭净资产负债表。
- 任务提醒系统。
- 风险强制阻断。

这些项会显著扩大边界,应在第一版稳定后再评估。
