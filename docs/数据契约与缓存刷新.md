# 数据契约与缓存刷新

本文档定义项目的本地数据契约。目标是让同步、指标、策略、回测、监控、页面展示都围绕同一套本地数据口径工作,避免不同模块各自解释字段。

## 1. 基本原则

- `backend/app/tickflow/repository.py` 是行情类 Parquet 的主要读写入口。
- 外部数据源只负责拉取和规范化,不直接承载页面或策略语义。
- `kline_daily_enriched` 磁盘表保持窄表;完整指标列由 Polars 在内存或查询时计算。
- DuckDB 视图服务冷查询、统计、元数据和扩展数据查询。
- Polars 缓存服务热路径:最新 enriched、instruments、指数维表、盘中聚合状态。
- 任一写入路径完成后,必须说明刷新哪些 DuckDB view、Polars cache 和前端 query。

## 2. 存储目录

| 目录 | 说明 | 主要写入者 |
| :--- | :--- | :--- |
| `data/kline_daily/` | 股票日 K 基础数据,按 `date=YYYY-MM-DD` 分区 | 日线同步、实时行情覆写 |
| `data/kline_daily_enriched/` | 股票 enriched 窄表,按日期分区 | 指标流水线、实时行情增量计算 |
| `data/kline_index_daily/` | 指数日 K 基础数据 | 指数同步 |
| `data/kline_index_enriched/` | 指数 enriched 窄表 | 指数同步后计算 |
| `data/kline_minute/` | 股票分钟 K | 分钟 K 同步 |
| `data/adj_factor/` | 除权因子 | TickFlow 除权同步 |
| `data/instruments/` | 股票标的维表 | 标的同步、AkShare 标的同步 |
| `data/instruments_index/` | 指数维表 | 指数同步 |
| `data/instruments_ext/` | 扩展维表数据 | 扩展数据服务 |
| `data/kline_ext/` | 扩展时序数据 | 扩展数据服务 |
| `data/financials/*/` | 财务指标、利润表、资产负债表、现金流 | 财务同步 |
| `data/depth5/` | 五档盘口 sealed 结果 | DepthService |
| `data/pools/` | 标的池缓存 | TickFlow pool 服务 |
| `data/screener_results/` | 策略结果缓存 | strategy_cache |
| `data/backtest_results/` | 回测结果 | 回测服务 |
| `data/user_data/` | 用户偏好、策略配置、告警、密钥覆盖等 | 设置、策略、监控服务 |

新增目录必须先说明:

- 数据是否按日期分区。
- 是否进入 DuckDB 视图。
- 是否需要 Polars 热缓存。
- 写入完成后如何失效前端 query。

## 3. 核心表字段

### 3.1 `instruments`

最小字段:

```text
symbol          统一股票代码,如 000001.SZ
name            股票名称
code            纯代码,如 000001
exchange        SH / SZ / BJ 或源字段映射
type            stock
listing_date    上市日期,可为空
industry        行业,可为空
total_shares    总股本,可为空
float_shares    流通股本,可为空
as_of           维表同步日期
```

约束:

- `symbol` 是唯一主键口径。
- 缺少 `float_shares` 时,换手率和市值相关字段允许为空或降级。
- 名称缺失时可由行情结果兜底补全,但不能改变 `symbol`。

### 3.2 `kline_daily`

最小字段:

```text
symbol
date
open
high
low
close
volume
amount
```

可选字段:

```text
turnover_rate
```

约束:

- 每个分区文件对应一个交易日。
- 同一 `symbol + date` 写入时使用 merge-upsert 或明确覆写策略。
- TickFlow 实时路径写入的是当天不复权行情;AkShare 阶段可写入前复权日线,但必须在 provider 文档中说明口径。

### 3.3 `kline_daily_enriched`

磁盘只持久化以下窄表字段:

```text
symbol
date
open
high
low
close
volume
amount
raw_close
raw_high
raw_low
turnover_rate
consecutive_limit_ups
consecutive_limit_downs
```

运行时可计算字段包括:

- 基础涨跌: `prev_close`, `change_pct`, `change_amount`, `amplitude`
- 均线/趋势: `ma*`, `ema*`, `macd_*`, `boll_*`
- 震荡/波动: `kdj_*`, `rsi_*`, `atr_14`, `annual_vol_20d`
- 量价/极值: `vol_ma*`, `vol_ratio_5d`, `high_60d`, `low_60d`, `momentum_*`
- 信号: `signal_*`
- join 字段: `name`, `total_shares`, `float_shares`

约束:

- 策略、筛选和回测可以依赖运行时 enriched 字段,但不能假设这些字段都已落盘。
- 新增指标字段应加入 `ENRICHED_COLUMNS`,并说明是否进入 `ENRICHED_STORAGE_COLS`。
- 改变 `change_pct`、复权、涨跌停、连板等核心字段口径时,必须写迁移说明或重算建议。

### 3.4 指数表

指数维表最小字段:

```text
symbol
name
code
asset_type=index
```

指数日 K 与股票日 K 字段相似,但不计算涨跌停、连板、股本、市值相关语义。

### 3.5 分钟 K

最小字段:

```text
symbol
datetime
open
high
low
close
volume
amount
```

约束:

- 分钟 K 是可选能力,不应影响日线、选股和基础回测闭环。
- AkShare 第一阶段不提供分钟 K。

### 3.6 扩展数据

扩展数据分为:

- `snapshot`:某日期或当前快照。
- `timeseries`:按日期或时间变化的数据。

必要约束:

- 配置必须记录 `id`, `label`, `mode`, `fields`, `updated_at`。
- 如果包含股票字段,必须在入库时做 `symbol` 规范化。
- 页面 join 扩展列时,不能污染策略核心缓存 key。

## 4. 写入和刷新责任

| 数据事件 | 推荐写入入口 | DuckDB view | Polars cache | 前端失效 |
| :--- | :--- | :--- | :--- | :--- |
| 股票维表写入 | `sync_instruments` / `akshare_sync.sync_instruments` | `instruments` | `repo.refresh_cache()` 或 `_refresh_instruments()` | 搜索、数据页、表格名称 |
| 股票日 K 写入 | `repo.append_daily()` / `repo.flush_live_daily()` | `kline_daily` | 无直接热缓存 | K 线、数据页 |
| 股票 enriched 写入 | `repo.append_enriched()` / `repo.flush_live_enriched()` | `kline_enriched` | latest enriched, live agg | 看板、策略、自选、监控 |
| 指数写入 | `repo.append_index_daily()` / `repo.append_index_enriched()` | index views | index instruments cache | 看板指数 |
| 分钟 K 写入 | minute sync service | `kline_minute` | 无 | 分时/K 线详情 |
| 扩展数据写入 | ext data service | ext views | 无 | 扩展页面、join 列表 |
| 告警写入 | monitor engine / alert store | 不进 DuckDB | 无 | 告警列表、未读徽标 |
| 策略配置变更 | strategy config service | 不进 DuckDB | strategy engine reload/cache | 策略页、回测参数 |

## 5. 变更检查表

改动数据模型或写入路径时,必须检查:

1. 是否新增或改变字段口径。
2. 是否需要更新 `ENRICHED_COLUMNS` 或前端 TypeScript 类型。
3. 是否影响已有 Parquet 兼容性。
4. 是否需要重算 enriched。
5. 是否刷新了对应 DuckDB view。
6. 是否刷新或清理了 Polars cache。
7. 是否触发前端 query invalidation 或 SSE 刷新。
8. 是否补充了 provider 降级行为。

如果一个改动无法回答以上问题,先补设计再实现。
